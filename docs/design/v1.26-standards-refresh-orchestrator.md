## What ships in v1.26 (C-26 `commands/standards-refresh.sh`)

The **missing orchestrator + periodic-refresh worker**. Three modes; idempotent on re-run.

### Mode A — Single-URL invocation

```
standards-refresh.sh \
    --url <url> \
    --target-namespace <ns> \
    --framework-id <id> \
    --source-id <sid> \
    --authoritative-publisher <name> \
    [--shape markdown-headings|numbered-list] \
    [--merge|--replace]         # default: --merge (preserve existing rule IDs by content_hash)
    [--out-dir <dir>]           # default: $CLAUDE_PLUGIN_ROOT/generated-code-quality-standards
    [--dry-run]
```

Per-URL pipeline:
1. **Extract (Stage 1)** — `extract-rules-from-url.sh --source <url> --shape <shape> --source-id <sid> --json` → segments array.
2. **Classify (Stage 2)** — for each segment: `classify-rule.sh --title <t> --prose <p> --json` → augmented segment with `{applies_to, applies_to_prose, confidence, signals}`. Low-confidence gets `needs_tier2_llm=true` flag; never dropped.
3. **Route (Stages 3+4)** — for each classified segment: `route-rule.sh --in <classified.json> --json` → augmented with `enforced_by[]`; auto-attaches `architectural-content` bundle when `applies_to_prose:true`.
4. **Assemble** — build a valid rule YAML: populated `source:` header (from CLI flags) + `rules[]` array (each rule has `id`/`name`/`description`/`detector`/`applies_to`/`enforced_by`/`provenance.section`/`content_hash`) + `recommended_set[]` + `all_set[]`.
5. **Merge-or-replace** — target file at `<out-dir>/<target-namespace>/<framework-id>.yaml`:
   - **Merge mode (default)** — read existing YAML; existing rule `id`s whose `content_hash` matches skip (preserved as-is); new rules (no matching `content_hash`) appended; rules present in existing but absent in new fetch → marked `deprecated: true` with `deprecated_at: <NOW>` reason (removed-upstream).
   - **Replace mode** — target file wholly overwritten.
6. **Write + touch last-fetch marker** — writes YAML atomically (temp file + rename) and updates `.claude-tdd-pro/<registry>-last-fetch/<source-id>.txt` with `NOW_ISO`.

### Mode B — Registry-iteration invocation

```
standards-refresh.sh \
    --registry .claude-tdd-pro/COMPLIANCE-URLS.yaml \
    [--now <iso>]               # default: current UTC
    [--force]                   # ignore fetch_frequency; refresh all entries
    [--dry-run]
```

Iterates every entry:
1. **Freshness check** — read `<registry>-last-fetch/<source-id>.txt`; skip if `NOW - last_fetch < fetch_frequency`. `--force` overrides.
2. **Namespace resolution** — per §17 G-9 mapping:
   - `COMPLIANCE-URLS` entry `jurisdiction: european-union` → `european-union/` (US Federal → `us-government/`, `international` → `industry-self-regulatory/`).
   - `STANDARDS-URLS` entry → `<inferred-namespace>` via id prefix or operator-set `source_namespace`.
   - `PR-SOURCES` entry `source_class: financial-industry` → `finance-industry/`.
3. **Framework-id** — the entry's `id:` field.
4. **Publisher** — the entry's `authoritative_publisher` (or `name` fallback).
5. **Shape** — the entry's `document-shape:` (optional; default `markdown-headings`; overridable via a per-entry field).
6. **Delegate to Mode A** per-entry.

### Mode C — Scheduler integration (invoked by C-19 / S-17 auto-refresh)

`compliance/auto-refresh-daily.sh` and `standards/auto-refresh-daily.sh` currently invoke a `--upstream-stub` placeholder. In a follow-on CL, they will invoke `standards-refresh.sh --registry <path> --now <iso>` in place of the stub — same output, real pipeline. This CL ships the orchestrator; the scheduler-wire is a one-line change in a subsequent CL.

## Idempotency + change-detection

- Every rule's `content_hash = sha256(title + "\n" + prose)` — deterministic across re-scrapes.
- Same URL content → same rule set → same YAML byte-identical (except `fetched_at` timestamp which is normalized-out per §31.9 A9 pattern).
- URL content changes:
  - **New rule appeared** — appended with `introduced_at: <NOW>`.
  - **Existing rule updated** (same title, different prose) — content_hash changes → treated as new (old deprecated).
  - **Rule removed from source** — marked `deprecated: true` with `deprecated_at: <NOW>` reason.
- Aggregator picks up the updated file → `active.json` reflects the fresh rule set on the next `standards-sync` run.

## Honest boundary

The orchestrator drives the deterministic Tier-1 pipeline. Per ADR-0009's discipline:
- **No clause silently dropped** — every extracted segment is classified; low-confidence gets `needs_tier2_llm=true` (never dropped).
- **Every rule cites its source** — populated `source:` header + `provenance.section` per rule.
- **No fabrication** — rules come from URL text via the deterministic segmenter.
- **Idempotent** — re-run produces same YAML (byte-identical modulo `fetched_at` timestamp normalization).
- **Change-detecting** — new/removed rules are marked explicitly (`introduced_at` / `deprecated_at`).

Network fetch happens inside `extract-rules-from-url.sh` (network-tolerant `curl` with 20s timeout).
Stage-1 today supports `markdown-headings` + `numbered-list`; HTML section extraction is a
CL-567 follow-on so URLs that publish as rendered HTML (eur-lex, sec.gov, hhs.gov) will
extract more comprehensively once landed.

The 6-stage pipeline is Tier-1 only in this CL: LLM (Stage 5 draft-custom-rule) and review-queue (Stage 6) integration are CL-566. Low-confidence rules are preserved with `needs_tier2_llm=true` so the LLM pass in CL-566 can find them.
