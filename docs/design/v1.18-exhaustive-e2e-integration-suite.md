## Why

The pipeline e2e specs `e2e-01..e2e-10` prove the **golden path** (scrape → tag → route → enforce →
audit) at a *representative* level — one rule per stage. The operator asked for **exhaustive**
integration coverage: every tool and every rule walked through the full pipeline, end to end.

This CL adds that exhaustive layer. It is integration testing that **composes** the existing
ADR-0008/0009 features (§28.29–§28.42); **no new feature ID, no new §2.X contract, no substrate
change** — only specs. The two genuinely-new population checks (per-tool runner sweep, routing
bijectivity, per-rule vocabulary fidelity) are embedded directly in the spec commands and reuse
the existing audit detectors where one already covers the invariant.

## Confirmation captured this CL: dependencies install when CTP is installed

`scripts/install.sh` **Step 8c** runs `rubric/runners/install-toolchain.sh` (backgrounded,
best-effort) during CTP `init`, provisioning the full 80-tool manifest. npm / pipx / cargo / go /
gem tools auto-install when their package manager is present; binary/manual tools print an
upstream URL for platform-specific install; copyleft `invoke_only` tools install unless
`--permissive-only`. The §28.28 hard-require policy then treats a missing **required** tool as a
broken install. GCTP inherits the toolchain by consuming CTP. So: **yes — all dependencies install
at CTP install time** (with the documented binary/manual-install caveat).

## Coverage map (10 specs)

Population facts: **80 tools** (`rubric/runners/toolchain.json`), **118 rules** across **66 files /
42 namespaces** (`generated-code-quality-standards/*/*.yaml`).

### Every tool (exhaustive over 80)

1. **every-tool-obeys-the-missing-tool-policy-through-the-runner** — iterates all 80 tools through
   `run-tool.sh` with `RUN_TOOL_FORCE_ABSENT=1`: required-absent → hard-fail (exit 1),
   optional-absent → not_enforced (exit 3). Proves every tool resolves + is wired into the runner +
   obeys §28.28. Deterministic (toolless-safe).
2. **every-tool-is-routable-and-every-routed-tool-resolves-to-the-manifest** — the routing table is
   bijective with the manifest: 0 orphan tools (every manifest tool reachable from some kind/bundle),
   0 ghost routes (every routed tool exists in the manifest). Deterministic.
3. **every-present-tool-runs-through-the-runner-to-a-sane-verdict** — opportunistic: every tool
   actually installed runs through `run-tool.sh` to a terminal verdict (0/1/3), never a usage error
   (exit 2). `timeout`-guarded; absent tools skipped → green on a toolless container.

### Every rule (exhaustive over 118)

4. **every-rule-maps-to-a-runnable-enforcer-with-parity-preserved** — reuses
   `audit-applies-to-parity.sh`: `rules=118 parity_fail=0 unrouted=0` (every rule routable, original
   detector preserved as first required binding).
5. **every-axis-tagged-rule-uses-only-canonical-four-axis-vocabulary** — every axis-scoped rule
   (112 of 118; the other 6 are axis-agnostic universal rules routed via their detector) tags only
   with vocabulary drawn from the Linguist / IaC / PURL authority mirrors — 0 invented kinds.
6. **the-whole-tool-and-rule-corpus-stays-commercially-sellable** — `audit-commercial-license.sh`
   green PLUS a per-rule check that every tool any rule binds to is FOSS with copyleft `invoke_only`.

### Full-pipeline golden paths per axis (scrape → tag → route → enforce → audit)

7. **pipeline-a-scraped-code-rule-flows-from-url-through-write-then-audit** — TypeScript rule:
   `extract-rules-from-url` → `classify-rule` (linguist `typescript`) → `route-rule` (eslint) →
   `composite-audit` passes a clean repo green.
8. **pipeline-a-scraped-iac-rule-from-url-is-audited-red-then-green** — Kubernetes rule: scrape →
   classify (iac `kubernetes`) → route (container scanner) → audit flags a privileged-pod repo red
   AND passes a compliant repo green.
9. **pipeline-a-scraped-prose-rule-routes-to-the-bundle-and-audits-markdown** — ADR rule: scrape →
   classify (`applies_to_prose`) → route (`architectural-content` bundle) → audit flags an ADR
   Markdown that leaves ingress unrestricted red.
10. **write-time-and-audit-time-enforce-the-same-rule-on-the-same-file** — the two-phase contract:
    a privileged-pod file blocked by the write-time per-file enforcer (`enforce-file.sh`) is the
    same file flagged red by the audit-time whole-tree gate (`composite-audit.sh`); a clean file
    passes both.

## Tool-independence

Specs 1, 2, 4–10 are deterministic (force-absent policy paths, static manifest/routing/corpus/mirror
data, and the dependency-free in-repo detectors for enforcement). Spec 3 is opportunistic
(`command -v` guard + `timeout`; only a usage-error exit-2 fails it), so the whole suite stays green
on a fresh toolless container while exercising live tools when present.
