## What was genuinely missing

Two connected gaps, both pre-committed by the architecture but not built:

1. **Registry-walker.** ADR-0009 line 50 names `standards-refresh.sh` as the driver that "scrapes source URLs". No such script existed (verified by grep across the whole plugin). The C-19 / S-17 schedulers invoked a `--upstream-stub` placeholder rather than a real fetch pipeline. Every registered URL sat on the shelf.
2. **Three of the five Stage-1 shapes ADR-0009 line 51 committed to.** `extract-rules-from-url.sh` shipped with 2 shapes (`markdown-headings`, `numbered-list`); ADR-0009 line 51 lists 5 (adds `html-sections`, `free-prose`, `pdf-sections`). Since nearly every registered URL is HTML, the missing shapes were the practical reason 0 rules materialized when tested empirically.

## What ships in v1.26 (Option 3 — hybrid)

**No new architecture invented.** Only existing infrastructure activated + shape coverage completed.

### 1. `commands/extract-rules-from-url.sh` extended to 5 shapes (per ADR-0009 line 51)

The Stage-1 command gains three new shapes alongside the existing two:

- **`html-sections`** — scans for `<h2..h6>` tags with `<h1>` treated as document title. Body is HTML-stripped prose from the heading tag until the next same-or-higher heading.
- **`free-prose`** — paragraphs (blank-line delimited); title = first policy-verb sentence (MUST / SHALL / MAY / SHOULD / REQUIRED) or first 8 words.
- **`pdf-sections`** — text-mode PDF (pdftotext-style output); segments by section labels (`Section 1.2`, `V14.3`, `§5.1`, `Article 5`, `Art. 6`).

Existing `markdown-headings` + `numbered-list` behavior is byte-identical. Same JSON output shape ({rule_id, title, prose, source, content_hash}).

### 2. `standards/fetchers/http-get.sh` — the generic upstream stub

The plugin's per-source `standards/fetchers/*.sh` scripts (`html-anchor.sh`, etc.) are SHAPE EXTRACTORS taking an already-downloaded local file. `standards/fetcher.sh` expects an `--upstream-stub <path>` executable that emits raw source content on stdout (verified at line 80). No source-specific network stubs existed. `http-get.sh` is a generic one: reads `URL` env, curls, emits stdout. Enables `standards/fetcher.sh` fragility-tier + cache-preservation for any URL without per-source stubs.

### 3. `commands/standards-refresh.sh` — the ADR-0009 line 50 registry-walker

The registry-walker orchestrator. Single mode (registry-iteration) — no invented "single-URL mode" that bypasses the fetcher-dispatch layer.

```
standards-refresh.sh --registry <path> [--force] [--now <iso>] [--dry-run] [--out-dir <dir>] [--replace]
```

Per-entry pipeline:

1. **Freshness-skip.** Check `<registry>-last-fetch/<source-id>.txt` vs entry's `fetch_frequency`; skip if within window. `--force` overrides.
2. **Fetch via existing dispatch.** `URL=<url> bash standards/fetcher.sh --source-id <sid> --fragility-tier <t> --strategy <s> --upstream-stub standards/fetchers/http-get.sh --cache <cache-dir> --emit-metadata <meta> --auto --no-confirm-default`. Honors §2.6 fragility contract; cache preserved on upstream failure.
3. **Extract per shape inferred from `fetcher:`.** Shape inference table: `html-anchor.sh` → `html-sections`; `markdown-headers.sh` → `markdown-headings`; `pdf-section.sh` → `pdf-sections`; `rfc-style.sh` → `free-prose`; other/absent → `markdown-headings` default. Invoke `commands/extract-rules-from-url.sh --source <cache-file> --shape <inferred> --source-id <sid> --json`.
4. **Classify + route.** Per segment, invoke `classify-rule.sh` (Stage 2) then `route-rule.sh` (Stages 3+4). Low-confidence gets `needs_tier2_llm=true` (never dropped per ADR-0009 "no clause silently dropped" contract).
5. **Assemble YAML with §28.40 `introduced_in`.** Populates `source:` header + `rules[]` (each with id / name / description / detector / applies_to / enforced_by / provenance.section / content_hash / `introduced_in: <NOW>` epoch tag) + `recommended_set[]` + `all_set[]`.
6. **Merge idempotently by content_hash.** Existing rules whose `content_hash` matches → preserved (their `introduced_in` frozen; §28.40 grandfathering). New rules → appended with `introduced_in: <NOW>`. Rules absent from fresh fetch → marked `deprecated: true` with `deprecated_at: <NOW>` reason `removed-upstream`.
7. **Atomic write** to `<out-dir>/<resolved-namespace>/<framework-id>.yaml` + update `<registry>-last-fetch/<source-id>.txt`.
8. **Namespace resolution per §17 G-9.** Compliance jurisdictions: `us-government` / `federal-financial-regulator` → `us-government/`; `european-union` → `european-union/`; `international` → `industry-self-regulatory/`; `gold-standard-process` → `linux-foundation/`. Standards entries: entry-declared `source_namespace` or `_universal` fallback. PR sources: `financial-industry` → `finance-industry/`.

## Honest boundary

The orchestrator drives the deterministic Tier-1 pipeline. Per ADR-0009 discipline:

- **No clause silently dropped** — every extracted segment is classified; low-confidence gets `needs_tier2_llm=true` (never dropped).
- **Every rule cites its source** — populated `source:` header + `provenance.section` per rule.
- **No fabrication** — rules come from URL text via the deterministic segmenter.
- **Idempotent** — re-run produces same YAML (byte-identical modulo `fetched_at` timestamp normalization); §28.40 `introduced_in` frozen when content_hash matches.
- **Change-detecting** — new rules get `introduced_at: <NOW>` (via `introduced_in`); removed rules get `deprecated: true` + `deprecated_at: <NOW>` + `deprecated_reason: removed-upstream`.

Network fetch happens inside `standards/fetcher.sh` (via `http-get.sh` upstream stub, curl `--max-time 20`). Stage 5 LLM (`draft-custom-rule.sh`) and Stage 6 (`review-queue.sh`) integration is CL-566 per ADR-0009 Wave-2. Compliance-side stubs at `compliance/fetchers/*.sh` are Wave-2 per-framework work; the orchestrator does not require them.
