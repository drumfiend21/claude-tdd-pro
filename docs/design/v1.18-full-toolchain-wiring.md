## Why

§28.32 provisioned a **16-tool** subset into `rubric/runners/toolchain.json` with four bespoke
adapters in `run-tool.sh` (eslint / markdownlint / cspell / checkov). The operator's canonical
architecture names an **exhaustive ~80-tool** inventory across all four axes (Linguist languages,
IaC dialects, PURL package types, K8s GVK) plus the cross-cutting families (secrets, SBOM,
supply-chain, accessibility, performance, rego, jsonschema) and the `architectural-content`
prose bundle. This CL wires the **whole** inventory and makes every routed tool actually run —
without writing 80 bespoke adapters.

This is a *capacity* expansion of two existing features (§28.31 `run-tool.sh`, §28.32
`toolchain.json` / `install-toolchain.sh`). **No new feature ID, no new §2.X contract.** It
composes §28.31 + §28.32 + the §28.28 hard-require policy + the §28.29 SARIF bus.

## What changed

### 1. `commands/promote-toolchain.js` — single source of truth (new)

A generator that emits BOTH `rubric/runners/toolchain.json` and
`standards/kind-to-tool-routing.yaml` from one table `T` of 80 tool tuples
`[tool, bin, installer, package, license, invoke_only, sarif_native, exec_mode, exec_args, kinds[]]`.
Routing is *derived* from each tool's `kinds[]` (no hand-edited YAML drift). Run:
`node commands/promote-toolchain.js`.

- **80 tools**, 42 SARIF-native, 9 copyleft `invoke_only` (semgrep LGPL, shellcheck/yamllint/
  hadolint/codespell/reuse GPL, slither/trufflehog AGPL, spotbugs LGPL).
- Installer kinds present in the table: `npm`, `pipx`, `cargo`, `go`, `gem`, `binary`, `manual`.
- Binary tools derive an `install_url` (`owner/repo` → GitHub releases) for the manual-install path.

### 2. Generic spec-driven runner — `rubric/runners/run-tool.sh`

The four proven bespoke adapters (eslint / markdownlint / cspell / checkov) are retained. Every
*other* tool runs through ONE generic path that reads the tool's `bin` + `exec` spec from the
manifest:

- `exec.mode: "sarif"` — the tool prints SARIF 2.1.0 to stdout (validated; results counted).
- `exec.mode: "sarif-file"` — the tool writes SARIF to `<outdir>/<exec.out>`.
- `exec.mode: "exit"` — exit-code mode: 0 = clean, non-0 = findings (SARIF synthesized from up to
  50 output lines).

So all ~80 tools are wireable with no per-tool shell code. The §28.28 missing-tool policy is
unchanged (present → run; required + absent → hard-fail exit 1; optional + absent → not_enforced
exit 3; `RUN_TOOL_FORCE_ABSENT=1` forces the absent path for deterministic toolless-container specs).
Unknown tool name → exit 2.

### 3. Advisory formatters (the correctness fix)

A pure **formatter** (`prettier --check`, `shfmt -d`) exits non-zero on any file that is not
byte-identical to its house style. That is an *auto-fixable style difference*, **not** a
correctness violation — e.g. valid JSON `{"ok":true}` is flagged only because prettier would add
spaces. Treating that as red produced a **false red on every well-formed-but-unformatted file**.

Fix: the generator tags pure formatters `advisory: true`. In exit-mode, when an advisory tool
exits non-zero, the runner records the differences as **`level: "note"`** findings but keeps the
blocking count at 0 and returns **exit 0 (`status=advisory`)** — the file is verified well-formed,
the formatter merely has style opinions. Linters / validators / security scanners stay blocking.

### 4. Green-if-verified verdict semantics (`composite-dispatch.sh`, `composite-audit.sh`)

With ~80 routed tools and most absent on a partial toolchain, the prior "incomplete unless every
tool ran" rule marked clean files `incomplete`. The verdict is now **green once the file is
VERIFIED** — the always-present dependency-free in-repo detectors ran clean AND/OR ≥1 present
routed tool ran clean — and `incomplete` only when *nothing* could verify it. Absent OPTIONAL
tools no longer degrade a file that something else verified. This preserves "never a vacuous
green" (the in-repo detectors always give a real verdict) while making a **partial toolchain
usable**.

- `composite-dispatch.sh`: adds `ngreen`; final verdict `red>0 → red`, else `green>0 → green`,
  else `not_enforced>0 → incomplete`, else green.
- `composite-audit.sh`: per-file `red`/`verified` tracking across enforce-file + dispatch +
  (markdown) bundle; `red → red`, else `verified → green`, else `incomplete`.

### 5. Installer coverage — `rubric/runners/install-toolchain.sh`

Added `cargo install`, `go install …@latest`, `gem install`, and an explicit `manual` arm
alongside the existing `npm` / `pipx` / `binary` arms, so the new installer kinds in the 80-tool
table produce real install plans (or a manual notice) instead of `failed reason=no-installer`.

## Verdict matrix (after this CL)

| file                          | enforce-file | routed tools            | verdict |
|-------------------------------|--------------|-------------------------|---------|
| valid `{"ok":true}`           | green        | prettier → advisory     | green   |
| broken `{ broken,, }`         | red          | (n/a)                   | red     |
| unformatted but valid source  | green        | formatter → advisory    | green   |
| real lint/security violation  | (varies)     | linter/scanner → red    | red     |
| clean file, all tools absent  | green        | all not_enforced        | green   |

## Spec sketch (`cl502-*`, 10 specs, tool-independent)

full inventory captured (≥80) · every tool has a runnable exec spec · routing binds json→formatter
and typescript→many linters · generic runner resolves+runs a manifest tool (opportunistic) ·
generic runner hard-fails required-absent · generic runner not_enforced optional-absent · generic
runner rejects unknown tool · formatter style difference is advisory not red · installer covers
cargo/go/gem · full inventory stays commercially sellable (copyleft all invoke_only).

## Composition / fidelity

- Composes §28.31 (`run-tool` / `composite-dispatch`) + §28.32 (`toolchain.json` /
  `install-toolchain`) + §28.28 (hard-require) + §28.29 (SARIF bus). No new feature ID / contract.
- Every tool open-source and free for commercial use; copyleft (GPL/AGPL/LGPL) `invoke_only`
  (arms-length subprocess, never bundled/redistributed). Verified green by
  `rubric/detectors/audit-commercial-license.sh` (80 tools, 0 violations).
