The user is regenerating AI-tool rule files from `CONVENTIONS.md` +
`QUALITY-BAR.md` so all the AI tools their team uses share one source
of truth.

## Pre-flight

1. **Confirm the source files exist:**
   - `${CLAUDE_PROJECT_DIR}/CONVENTIONS.md` (project-level conventions)
   - `${CLAUDE_PROJECT_DIR}/QUALITY-BAR.md` (project-level standards;
     fall back to `${CLAUDE_PLUGIN_ROOT}/QUALITY-BAR.md` if absent)

   If `CONVENTIONS.md` is missing, OFFER to scaffold from
   `${CLAUDE_PLUGIN_ROOT}/templates/CONVENTIONS.md`. Do not generate
   sister files until the source exists.

2. **Refuse to overwrite hand-edited files.** Each generated file
   starts with the marker:
   ```
   <!-- GENERATED by claude-tdd-pro /sync-rules. Edit CONVENTIONS.md instead. -->
   ```
   If a target file exists WITHOUT this marker on its first
   non-blank line, STOP. Surface the path. Ask the user to either:
   - move the existing file aside (`mv .cursor/rules/conventions.mdc .cursor/rules/conventions.local.mdc`), then re-run; OR
   - explicitly authorize overwrite by re-running with `force=true`.

## Targets to generate

Each target reads CONVENTIONS.md + QUALITY-BAR.md and emits the
tool-specific format. Where token budget is constrained (Copilot,
Cursor "always-apply") prefer condensation over completeness.

### Cursor

Path: `.cursor/rules/conventions.mdc`

```
---
description: Project conventions (auto-synced from CONVENTIONS.md by claude-tdd-pro)
alwaysApply: true
---
<!-- GENERATED by claude-tdd-pro /sync-rules. Edit CONVENTIONS.md instead. -->

[Tech stack, Style, Anti-patterns sections — condensed to ≤2000 tokens]
```

Also emit per-area glob-scoped rules under `.cursor/rules/<area>.mdc`
if `${CLAUDE_PROJECT_DIR}/conventions/<area>.md` exists. Each such
file gets `globs:` from a header in the source.

### GitHub Copilot

Path: `.github/copilot-instructions.md`

```markdown
<!-- GENERATED by claude-tdd-pro /sync-rules. Edit CONVENTIONS.md instead. -->

# Project Instructions for GitHub Copilot

[Tech stack, Build & run, Style, Anti-patterns — Copilot's
recommended sections, capped at ~2 pages per Copilot best
practices]
```

### Aider

Path: `.aider.conf.yml` — patch (do not overwrite). Add or update:

```yaml
read:
  - CONVENTIONS.md
  - QUALITY-BAR.md
```

If the file already exists, MERGE the `read:` list (don't clobber
other Aider config).

### Windsurf

Path: `.windsurfrules`

```
<!-- GENERATED by claude-tdd-pro /sync-rules. Edit CONVENTIONS.md instead. -->

[Same content as Cursor's, since Windsurf uses similar token budget]
```

### AGENTS.md (cross-tool standard)

Path: `AGENTS.md`

```markdown
<!-- GENERATED by claude-tdd-pro /sync-rules. Edit CONVENTIONS.md instead. -->

# Project guide for AI coding agents

[Full content of CONVENTIONS.md, plus a short pointer to
QUALITY-BAR.md for review standards.]

This file follows the AGENTS.md cross-tool convention used by
Codex / Jules / Amp and is read by any agent harness that supports it.
```

### Continue.dev

If `${CLAUDE_PROJECT_DIR}/.continue/` exists, also emit
`.continue/rules/conventions.md` mirroring the Cursor content.

## Process

1. Read CONVENTIONS.md (required) and QUALITY-BAR.md (required).
2. For each target above, check if the target file exists:
   - if missing: emit a fresh GENERATED file
   - if exists with the GENERATED marker: regenerate
   - if exists WITHOUT the marker: STOP per the pre-flight rule
3. After emitting, list:
   - files created
   - files updated
   - files skipped (with reason)
4. Suggest: `git add -A` to track the generated files, OR add them
   to `.gitignore` if the team prefers each developer to regenerate
   locally.

## What NOT to do

- **Do not** invent content not present in CONVENTIONS.md or
  QUALITY-BAR.md. The point is one source of truth — adding tool-
  specific extras defeats the purpose.
- **Do not** edit CONVENTIONS.md from this command. It is the source.
- **Do not** silently overwrite hand-edited rule files.
