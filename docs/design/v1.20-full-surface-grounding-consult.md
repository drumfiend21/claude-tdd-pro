## The invariant

Everything CTP produces for a consumer MUST be reasoned against the **full rule/namespace surface**
(118 rules / ~42‚Äì43 namespaces) at **architecture-production time** ‚Äî not against the ~18-source
cloud subset (AWS-WA / NIST-800-53 / SRE / OTel / OWASP / Fowler / ‚Ä¶) that the production chain used.

## The composition gap (confirmed at pin `a69f380`)

- `rubric/aggregator.sh` (G-5) already walks `generated-code-quality-standards/` and builds the FULL
  surface: **118 rules**, top-level `namespaces_seen`, each rule carrying `source_namespace` +
  `provenance[]`.
- But the production chain ‚Äî `commands/business-translate.sh` ‚Üí `commands/architect-recommend.sh`
  (‚Üí `architect-session.sh`) ‚Äî **never references `aggregator.sh` or `generated-code-quality-standards/`**.
  `business-translate.sh` grounds against a **hardcoded 18-source `source_id` set**; `architect-recommend.sh`
  derives its `grounding` purely from those translated concerns.
- Net: a produced design reasons against ~18 cloud sources ‚Üí **6 of 42 namespaces consulted, 36
  un-consulted** (measured). 36 code/config namespaces (typescript, react, node, owasp, jwt, iam,
  ansible, azure, cfn, k8s, ‚Ä¶) are invisible at architecture-production time.

## The additive fix ‚Äî S-56 / contract ¬ß2.34

`commands/full-surface-consult.sh` ‚Äî the consult engine that **ingests the aggregator's full surface**
(the composition the chain lacked) and measures a produced design against **every** namespace:

- `--design <technical-requirements.json>` (concerns carry `source_id` grounding);
  `[--surface <aggregator.json>]` (default: run `aggregator.sh`); `[--require-complete]`; `[--json]`.
- A namespace is **consulted** iff the design grounds against ‚â•1 source that a rule in that namespace
  cites (`provenance[].source`); otherwise it is surfaced as **`needs_grounding`** ‚Äî **cite-or-decline**,
  never silently omitted.
- Marker: `full-surface-consult rules_total=<r> namespaces_total=<n> consulted=<c> needs_grounding=<u>
  status=<complete|incomplete>`; per un-consulted namespace `consult namespace=<ns> status=needs_grounding`.
- `--require-complete` exits 1 when any namespace is un-consulted ‚Äî the **Stage-5 verdict-completeness
  gate** (GCTP TICKET-113).

This closes the **composition** gap (the chain now ingests + measures against the full surface) and
provides the gate. Driving `needs_grounding ‚Üí 0` ‚Äî i.e. broadening `business-translate`/`architect-recommend`
to emit concerns grounded across all namespaces ‚Äî is the follow-on the ¬ß2.34 contract now mandates.

## ¬ß2.34 contract (new)

**Full-surface production-grounding contract.** Every architecture/design CTP produces for a consumer
MUST be reasoned against the full aggregated rule surface (`rubric/aggregator.sh`, 118 rules / all
namespaces) at production time; a namespace the design does not reason against is surfaced as
`needs_grounding` (cite-or-decline), never silently omitted. Enforced by `commands/full-surface-consult.sh`
(deterministic; `--design`, exit 0 complete / 1 incomplete under `--require-complete` / 2 usage).
