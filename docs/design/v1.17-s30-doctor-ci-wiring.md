## What

A new `--check cloud-conventions --root <dir>` arm in `commands/doctor.sh`. It
discovers IaC under the root, maps each file to its tool ruleset, and runs the
existing S-30 engine, aggregating to the S-30 exit-code contract:

- **Discovery (no false positives):** `*.tf` â†’ terraform; `*.bicep` â†’ bicep;
  `*.json` / `*.yaml` / `*.yml` are treated as CloudFormation **only** if they carry
  the `AWSTemplateFormatVersion` marker. Ordinary JSON/YAML (package.json, configs)
  is never mistaken for IaC. Discovery is recursive.
- **Safe-by-default (the no-regression property):** a repo with **no** cloud IaC is a
  no-op â€” `cloud-conventions no_cloud_iac=true status=skipped`, **exit 0**. This is
  what makes wiring into `/doctor` safe for the vast majority of repos (which have no
  Terraform/Bicep/CFN): the new check can never fail a non-cloud repo.
- **Green:** all discovered IaC clean â†’ `status=green files=<n> convention_violations=0`,
  exit 0.
- **Red:** any drift â†’ per-file `file=<path> tool=<t> status=red convention_violations=<n>`
  and a summary `status=red files=<n> red=<r> convention_violations=<m>`, **exit 1** â€”
  the same 0-green / 1-red contract as `cloud-conventions.sh`. Each underlying violation
  is still grounded (cite-or-decline) by the S-30 engine.
- **Usage:** missing/invalid `--root` â†’ exit 2 (matches the `validate-all` / `coverage`
  arms).

## Additivity / no-regression guarantees

- The change is a **new `case` arm** inserted before the `*)` unknown-check
  fall-through. No existing arm, flag, or default is modified. Every prior `--check`
  invocation behaves identically.
- It reuses the already-parsed `--root` flag; no new CLI parsing, no new global state.
- The engine it calls (`cloud-conventions.sh`) is unchanged.
- Because absence-of-IaC is a green no-op, adding this check to any aggregate
  `/doctor`/CI run cannot turn a previously-green non-cloud repo red.
